# 🔄 도서 관리 시스템 (Book Management System) 기능 명세서

본 문서는 Spring Boot 백엔드와 Next.js 프론트엔드 연동을 기반으로 구현된 도서 관리 프로그램의 핵심 기능 명세입니다. 처음 연동을 구현하며 학습한 데이터 흐름과 예외 처리 내용을 간결하게 정리했습니다.

---

## 🛠️ 1. 도서 기본 관리 기능 (CRUD)

### ① 새 도서 등록 (POST `/api/books/`)
* **기능:** 사용자가 입력한 제목, 저자, 가격을 수신하여 데이터베이스에 저장합니다.
* **특징:** 도서 등록 시 초기 대여 가능 여부(`available`) 상태값은 시스템 기본값인 `true`(대여 가능)로 자동 설정됩니다.

### ② 도서 목록 조회 (GET `/api/books/`)
* **기능:** 현재 등록된 모든 도서 리스트를 메인 화면에 출력합니다.
* **특징:** 백엔드 응답 데이터를 프론트엔드 카드 컴포넌트에 바인딩하여 제목, 저자, 가격, 대여 상태를 시각적으로 보여줍니다.

### ③ 도서 상세 정보 조회 (GET `/api/books/{id}`)
* **기능:** 도서 고유 번호(ID)를 기반으로 해당 도서의 상세 정보를 단건 조회합니다.
* **특징:** Next.js의 동적 라우팅(`books/[id]`)을 활용해 경로 매개변수를 추출한 뒤 백엔드에 요청하며, 없는 ID인 경우 404 에러 페이지를 처리합니다.

### ④ 도서 정보 삭제 (DELETE `/api/books/{id}`)
* **기능:** 선택한 도서 정보를 데이터베이스에서 완전히 영구 삭제합니다.
* **특징:** 백엔드 삭제 성공 신호(`204 No Content`)를 수신하면 클라이언트 화면에서도 해당 도서 데이터를 실시간으로 제외합니다.

---

## 🔄 2. 도서 대여 및 반납 상태 변경 (PUT `/api/books/{id}`)

상세 페이지에서 버튼을 클릭하면 현재 도서 상태를 확인하고 비동기 통신을 통해 대여 가능 여부(`available`)를 정반대로 전환(토글)하는 가산점 구현 기능입니다.

### ① 상태 전환 시나리오
* **대출 처리 (`true` ➡️ `false`)**
  * 현재 상태가 `true` (대여 가능 / 🟢 초록 배지)일 때 버튼 클릭 시 역전값인 `false`를 페이로드에 실어 백엔드로 전송 ➡️ DB 반영 후 🔴 **'대여 중'** 상태로 실시간 전환
* **반납 처리 (`false` ➡️ `true`)**
  * 현재 상태가 `false` (대여 중 / 🔴 빨간 배지)일 때 버튼 클릭 시 역전값인 `true`를 페이로드에 실어 백엔드로 전송 ➡️ DB 반영 후 🟢 **'대여 가능'** 상태로 실시간 리턴

### ② 주요 트러블슈팅 및 예외 처리 내용
* **필수 데이터 유실 방지 (Data Guard):** 대여 상태(`available`) 수정 시, 제목이나 저자 같은 필수 필드가 `null`로 전송되어 DB 제약 조건(`NOT NULL`) 위반 에러가 나는 것을 막기 위해 기존 데이터를 안전하게 보존(Guard)하여 함께 전송하도록 설계했습니다.
* **직렬화 규칙 명시:** 자바 롬복의 `boolean` Getter 규칙(`isAvailable`)과 프론트엔드 JSON 키(`available`) 간의 매핑 오류를 백엔드 DTO 내 `@JsonProperty("available")` 선언을 통해 해결했습니다.
* **화면 동기화 (Revalidation):** `PUT` 수정 요청 성공 즉시 클라이언트단에서 `window.location.reload()`를 트리거하여 Next.js 서버 컴포넌트가 최신 DB 적재 데이터를 다시 긁어오도록 강제함으로써 화면 배지를 즉각 동기화합니다.
